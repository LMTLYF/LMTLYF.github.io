最近使用github+hexo+node.js在搭建个人的博客，发现与git相关的知识总是零零散散的，这一篇文章来总结一下我对git所有的了解，如果有错误的地方，大家可以指出来，我会不断完善。

了解git,github

git,github是什么

git:是一个分布式版本控制系统,版本控制是一种记录一个或若干文件内容变化，以便将来查阅特定版本修订情况的系统。

GitHub：是用于版本控制和协作的代码托管平台，GitHub 可以托管各种Git版本库，并提供一个web界面，GitHub上的项目都存储在仓库中，后者包含与项目相关联的一切：代码，项目参与者的信息，问题和bug报告等
简单来说Github是代码托管平台，是协作的工具;而Git是版本控制工具，是可以将你的代码上传到github的工具

安装git

windows：到git官网上下载,Download git,下载后会有一个Git Bash的命令行工具，以后就用这个工具来使用git。

linux：对linux来说实在是太简单了，因为最早的git就是在linux上编写的，只需要一行代码

sudo apt-get install git

安装好后，可以来查看一下版本

git --version

解释Git CMD,Git GUI,Git Bash

安装之后我们发现会出现以下三个东西

（1）Git CMD：Git中的Bash是基于CMD的，在CMD的基础上增添一些新的命令与功能。所以建议在使用的时候，用Bash更加方便

（2）Git GUI：Git GUI是Git Bash的替代品，他为Windows用户提供了更简便易懂的图形界面（相当于客户端但是并没有什么用）

（3）Git Bash是命令行操作，官方介绍有一句就是“让linux用户感到宾至如归”，安装完成后再桌面右键就可以看到它了，它的命令和linux命令几乎一模一样

github的客户端操作

形象理解github

在介绍客户端操作之前，根据b站的官方视频还有我自己的了解（或许不太准确←_←），github有点类似于一个大型论坛，我用一个事例来解释一下为什么这样说，农民在割草，发现割草机的效率太低，他可以去一个技术员的下提出问题，技术员看到问题就开始进行这个项目，并把代码放到（仓库）中，唉，好巧不巧，有其他的技术员路过，可以直接克隆代码到自己的仓库中进行修改：技术员也可以把这个项目让他的同伴加入一起做，最后他们做好了，技术员觉得：“哇，做的真不错”，就可以上传到最初的那个代码仓库中。在了解了这样一个实例之后，客户端最重要的操作就了解的差不多了。接下来在完整的关于github的操作中，你会更加了解客户端的一些名词。

github的整体操作

整体操作中完完全全从头介绍了如何使用（对新手小白极其友好）

验证身份

本电脑想要与github连接必须先验证身份

git config --global user.name "yourname"
git config --global user.email "youremail"

检验是否输入成功：

git config user.name
git config user.email

配置SSH

配置了SSH之后才可以把本地的文件通过git成功上传到github

创建仓库

（1）在本地先创建一个文件夹（相当于本地仓库）,右键文件夹打开git bash，输入

$ git init

对本地仓库进行初始化，在该文件夹中会生成.git的隐藏文件，如果看不见就把显示中的显示隐藏文件打开

（2）在github上创建仓库
个人主页必须要和用户的GitHub帐号同名，所以每个用户有且只能有一个repo作为个人主页，且必须是.github.io的形式
项目主页没啥要求

（3）SSH连接（本地仓库和github创建的仓库连接）
在本仓库的code找到，进行复制
在你的本地仓库打开gitbash

git remote add + 名字 + 连接地址（连接地址就是刚才复制的）

测试是否成功

git remote -v

显示这样就是成功了
如果以后不想连接了

git remote remove + 名字

文件上传

（1）git add 将修改的文件添加暂存区

$ git add +文件名.文件类型 ，将某个文件加到缓存区
$ git add +文件名.文件类型 ... 文件名.文件类型 ，将n个文件添加到缓存区
$ git add xx文件夹/*.html，将xx文件夹下的所有的html文件添加到缓存区。
$ git add *hhh ，将以hhh结尾的文件的所有修改添加到暂存区
$ git add Hello* ，将所有以Hello开头的文件的修改添加到暂存区
git add -u ，提交被修改(modified)和被删除(deleted)文件，不包括新文件(new)
git add .，提交新文件(new)和被修改(modified)文件，不包括被删除(deleted)文件
…

git add -A，提交所有变化。git add前几条都可以记不住，这个必须记住！！

（2）git commit可以简单的理解为给你刚才add的东西加一个备注，你上传到远程仓库之后，修改的文件后边会显示这个备注

$ git commit -m '修改注释'（如果没有m直接就进入vim页面）

（3） git push激动的搓搓小手，终于要把文件推到远程仓库了

$ git push -u 仓库名称 分支（第一次-u，之后直接git push）

仓库名称：注意是本地的仓库名，并不是github中的
分支：你现在只有主分支，所以分支直接写master。以后合作项目的时候，成员之间建了不同的分支，你就可以往你自己的分支上推。
问题：还有一个$ git push origin master -f 强制推送，如果你某次推送失败，git bash报错，你懒得处理错误，你就可以用这个。但是有风险，因为报错90%是因为你本地仓库和远程仓库数据发生冲突，使用这个会直接用本地数据覆盖掉远程数据，可能损失数据哦。

文件上传错误的一些处理

（1）查看提交记录

git log

（2）出现一些错误，自己简单修改

git add -A
git commit -m “一些有关操作的描述” 
git push 本地仓库名 master

如果推送不上的话，可以强制推送（毕竟自己很清楚自己改的是什么，应该是对整个项目没有太大影响的一些问题，比如命名错了或者注释错了）

git push origin master -f 这个-f就是force

修改注释，并且不会有过程的显示

git commit --amend

（3）出现错误时有团队合作，强制提交会损坏队友的代码,直接在客户端修改比较好一些，或者把代码拉下来，进行修改
第一种拉代码方法

$ git pull 本地仓库名称（直接进到客户端编辑，pull展示）

第二种拉代码方法

pull = fetch+merge
git fetch 将数据拉下来，但是没修改本地的commit和文件
git merge 改变本地数据

文件克隆

仓库是你自己的，你就使用SSH连接，不是你自己的，你没权限你就切换到HTTPS，再复制地址。它克隆下来是一个文件夹，你想把文件夹放哪里就在哪打开gitbash

$ git clone 加上你刚才的地址（注意这时候连接的仓库是别人的）

一些删除操作

（1）删除无用的仓库：直接到危险区域删掉
（2）本地和远程都删除文件

$ git add * //把本地仓库的文件上传到缓存。
$ git commit -m 'del' //把第一步上传到缓存的东西上传到本地仓库，其中'del'是操作标识，内容随便填，方便用户观看。
$ git push origin master //把本地仓库的文件上传到远程仓库。

（3）只删除远程仓库
``
$ git pull origin master //将远程仓库里面的项目拉下来。
