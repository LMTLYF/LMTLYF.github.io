关机，重启

关机

shutdown -h now //立刻关机
shutdown -h 5 //5分钟后关机
poweroff //立刻关机

重启

shutdown -r now //立刻重启
shutdown -r 5 //5分钟后重启
reboot //立刻重启

帮助

help

命令 --help

man

man 命令

目录操作

切换目录

cd ~ //切换到home目录
cd- //切换到上次访问的目录

目录图

重命名，复制文件

mv

mv 源目录 目标目录

注意：这是移动一个文件，只有一个；移动到了目标目录之下，所以目标目录要存在

cp

cp -r 源目录 目标目录

注意：这是复制文件，有两个文件；也是在目标目录之下（如果是要改名的话，要复制最底部的那一个）

文件（增删改查）

新建

touch 文件名 //新建了一个空文件
vim 文件名 //如果没有也就新建了一个文件

删除

rm -rf

修改

这里是vim的常用命令

dd //删除当前行
5dd //删除包括当前及以下共5行
yy //复制当前行
5yy //复制当前及以下共5行
p //粘贴，粘贴的时候是在该光标的下面

查看

cat //只显示了最后一页
more //回车下一行，空格下一页，q结束查看
less //PgUp,PgDn翻页，q结束查看
tail -n //指定了是结尾的几行，ctrl+c结束
head -n //指定了是开头的几行，ctrl+c结束

压缩和解压缩

压缩

tar -zcvf 压缩后文件名 要压缩文件名 //压缩，解压也是解压到了某文件夹的下面

解压缩

tar -zxvf 要解压文件名 -C 解压后文件名 //解压

重定向

>：将本来出现在屏幕的标准输出信息重定向到一个文件中
>>：实现重定向时不覆盖原有内容，而是在文件末尾追加内容

软硬链接

软链接

软链接（符号链接）：相当于电脑上的快捷方式，B是A的软链接，A消失了B无效，B消失了对A并没有影响

ln -s 文件 软链接的名字

硬链接

硬链接：有点类似于复制，主要是防止误删，B是A的硬链接，只有二者都被删了，文件才会消失

ln 文件 硬链接的名字

文件操作

查看文件详细属性

ls -al //查看所有的文件，还有以.开头的隐藏文件
ls -l //查看所有文件，不过没有隐藏文件

理解字符含义

第一个：
d:目录
-:文件
l:链接文档
b:装置文件里面的可供储存的接口设备(可随机存取装置)
c:装置文件里面的串行端口设备，例如键盘、鼠标(一次性读取装置)
剩余的（三个一组）：
分别是文件所有者，文件所有者所在的组中其他用户，其他用户

修改文件所属主

chown [–R] 属主名 文件名 //-R是下面的也都改了

修改文件所属组

chgrp [-R] 属组名 文件名

修改文件权限

数字进行修改

r:4  w:2  x:1
owner = rwx = 4+2+1 = 7
group = rwx = 4+2+1 = 7
others= --- = 0+0+0 = 0
chmod -R 770 文件或目录 //加了R连目录下面的都一起更改了

字符进行修改

user(u)：所有者 group(g)：组 others(o)：其他 
直接进行+ - = 操作
chmod -R u=rwx,g=rx,o=r  文件或者目录

理解用户和组

多用户多组

linux本来就是多用户多任务的操作系统，其他人可能会远程使用你的电脑，这时候我们就可以创建一个用户让他来登录；根据该用户的作用我们可以把它分到不同的组

查看用户

注意：不能通过命令来看，自能看文件
用户（user）相关的配置文件；
/etc/passwd //用户（user）的配置文件；
/etc/shadow //用户（user）影子口令文件；
/etc/passwd

用户名:口令:用户标识号:组标识号:注释性描述:主目录:登录Shell
口令：其实就是密码，为了安全这里用x来表示
用户标识号：可能多个用户会有一样的标识号，但是它们的主目录和登录shell应该不同
登录shell：默认是nologin
/etc/shadow

用户名：密码加密：密码已经使用的日期（1970-01-01开始）：密码多少天后可以修改：多少天后必须修改：修改之前几天提醒：未修改延长几天：时间过期：保留字段

查看组

用户组（group）相关的配置文件；
/etc/group //用户组（group）配置文件；
/etc/gshadow //用户组（group）的影子文件；
/etc/group

组名:口令:组标识号:组内用户列表（多个的用户用，隔开）
/etc/gshadow

用户组：用户组密码：用户组管理者：组成员（多个成员用，隔开）
用户组密码：如果是空的或有!，表示没有密码；
用户组管理者：这个字段也可为空，如果有多个用户组管理者，用,号分割；
组成员：如果有多个成员，用,号分割；

用户相关操作

新建用户

useradd 选项 用户名 （选项的出现其实和其配置文件有很大关系的）
-c comment 指定一段注释性描述
-d 目录 指定用户主目录，如果此目录不存在，则同时使用-m选项，可以创建主目录。
-g 用户组 指定用户所属的用户组。
-G 用户组，用户组 指定用户所属的附加组。
-s Shell文件 指定用户的登录Shell。
-u 用户号 指定用户的用户号，如果同时有-o选项，则可以重复使用其他用户的标识号。

删除用户

userdel 选项 用户名 （常用选项-r 直接将其主目录也删除）

修改用户

usermod 选项 用户名 （和添加用户的选项一样）

密码（口令）管理

passwd 选项 用户名
-l 锁定口令，即禁用账号。
-u 口令解锁。
-d 使账号无口令。
-f 强迫用户下次登录时修改口令

组相关操作

新建用户组

groupadd 选项 用户组
-g GID 指定新用户组的组标识号（GID）。
-o 一般与-g选项同时使用，表示新用户组的GID可以与系统已有用户组的GID相同

删除用户组

groupdel 用户组

修改用户组

groupmod 选项 用户组
-g GID 为用户组指定新的组标识号。
-o 与-g选项同时使用，用户组的新GID可以与系统已有用户组的GID相同。
-n 将用户组的名字改为新名字

服务

服务(service) 本质就是进程，但是是运行在后台的，不受前台和终端的控制，周期性的执行一些事情，比如(mysql , sshd 防火墙等)，因此我们又称为守护进程。
查看服务

查看服务

/etc/init.d目录下有众多的服务
service --status-all //可以查看服务的运行状况

管理服务

service 服务名 start|stop|restart|reload|status

进程

进程就是当前正在执行的一些程序

查询进程

ps -aux


USER: 进程拥有者
PID: pid
%CPU: 占用的 CPU 使用率
%MEM: 占用的记忆体使用率
VSZ: 占用的虚拟记忆体大小（virtual）
RSS: 占用的记忆体大小（true）
TTY: 终端的次要装置号码 (minor device number of tty)
STAT: 该行程的状态:
D: 无法中断的休眠状态 (通常 IO 的进程)
R: 正在执行中
S: 静止状态
T: 暂停执行
Z: 不存在但暂时无法消除（僵尸状态）
W: 没有足够的记忆体分页可分配
<: 高优先序的行程
N:低优先序的进程
L：有记忆体分页分配并锁在记忆体内（实时系统或AI/O）
START:行程开始时间
TIME:执行的时间
COMMAND:所执行的指令

以树状图查看

pstree -u //以用户来查看
pstree -p //以id来查看

实时显示进程

top -选项
选项
d：指定每两次屏幕信息刷新之间的时间间隔。当然用户可以使用s交互命令来改变之。
p:通过指定监控进程ID来仅仅监控某个进程的状态。
q:该选项将使top没有任何延迟的进行刷新。如果调用程序有超级用户权限，那么top将以尽可能高的优先级运行。
S：指定累计模式。
s：使top命令在安全模式中运行。这将去除交互命令所带来的潜在危险。
i：使top不显示任何闲置或者僵死进程。
c:显示整个命令行而不只是显示命令名。
交互指令
top命令进入之后可以进行的一些操作命令
p以CPU使用率排序（默认）
M以内存使用率排序
N以PID排序
q退出

终止进程

kill -选项 进程号（-9强制）
killall 进程名称
